# Generated artifacts

## File layout

```
specgen/
  source/                       <- authored inputs
    noc_function_blocks.json    <- feature inventory (not consumed by codegen)
    noc_function_blocks.schema.json
    constants.yaml              <- params domain SSoT
    interface_handshake.json
  generated/
    json/                       <- JSON SSoT (packet / signals)
      ni_packet.json
      ni_packet.schema.json
      ni_signals.json
      ni_signals.schema.json
      ni_protocol_rule_index.json
      ni_protocol_rule_index.schema.json
    cpp/                        <- C++ headers (tracked, drift-gated)
      ni_flit_constants.h
      ni_signals.h
      ni_params.h
    sv/                         <- SV packages (tracked, drift-gated)
      ni_flit_pkg.sv
      ni_signals_pkg.sv
      ni_params_pkg.sv
```

`generated/cpp/` and `generated/sv/` are tracked in git. `codegen.py --check` (and the build's `codegen_check` target) gate drift between the committed artifacts and the SSoT sources.

## Provenance banner

Every elaborated file starts with five comment lines plus a horizontal rule above and below:

```
// ----------------------------------------------------------------------------
// AUTO-GENERATED by tools/codegen.py -- DO NOT EDIT
// Source:    specgen/generated/json/ni_packet.json
// Source SHA: ea56f053b2a8
// Generator version: v1.0.0
// Generated at: 2026-05-27T06:21:43Z
// ----------------------------------------------------------------------------
```

| Banner field | Meaning |
|--------------|---------|
| `Source` | POSIX path to the JSON SSoT this file was elaborated from, prefixed with `specgen/`. |
| `Source SHA` | First 12 hex chars of the SHA-256 of the source JSON bytes. |
| `Generator version` | Tool version, hard-coded in `tools/elaborate/common.py`. Independent of the spec version. |
| `Generated at` | UTC ISO-8601 timestamp. Excluded from `--check` comparison. |

`sv` outputs use the same banner with `//` as the comment prefix.

## What each artifact contains

### `ni_flit_constants.h` and `ni_flit_pkg.sv`

- Top-level widths: `FLIT_WIDTH`, `HEADER_WIDTH`, `PAYLOAD_WIDTH`, `LINK_WIDTH`, `FLIT_DATA_WIDTH`, `HEADER_DATA_WIDTH`, `WSTRB_WIDTH`.
- `namespace header`: per-field `<NAME>_LSB`, `_MSB`, `_WIDTH`, `_ENABLED`.
- `namespace payload`: per-channel `<CH>_WIDTH` for `AW`, `AR`, `W`, `B`, `R`.
- `namespace width`: every named width referenced by the spec.
- `static_assert` invariants such as `FLIT_WIDTH == HEADER_WIDTH + PAYLOAD_WIDTH` and the SECDED bound.

### `ni_signals.h` and `ni_signals_pkg.sv`

- Per-interface, per-channel pin-level signal declarations.
- `<PIN>_RESET` constants for outputs whose reset value is fixed.

## Drift policy

After a JSON SSoT changes:

1. `py -3 specgen/tools/codegen.py --check` will exit 1 listing any artifact that needs refresh.
2. Re-elaborate the affected `(target, domain)` and commit alongside the SSoT change.

`--check` is the canonical contract: if it passes, downstream code and RTL match the JSON SSoT exactly (timestamp aside).
